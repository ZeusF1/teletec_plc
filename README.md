# OSBB PLC Meter Management

Система управління PLC-лічильниками для ОСББ.

## Призначення

Розроблена для ОСББ (об'єднань співвласників багатоквартирних будинків) що використовують PLC-лічильники MTX (MTX1, MTX3) підключені через маршрутизатор RT4 (STM32 OpenSTLinux).

**Проблема:** Штатний процес PollReader обходить ~528 лічильників циклічно (~2 за цикл, ~2 хв/цикл), збираючи лише свіжі дані. Глибокий архів (за минулі місяці) отримати неможливо — PollReader скидає дату після обробки.

**Рішення:** Модуль прямого зчитування `meter_archive.php` — надсилає PLC-команди напряму через маршрутизатор, минаючи чергу PollReader. Дозволяє отримати будь-які архівні дані з будь-якого лічильника за вказаний період.

---

## Типи даних

- **LP** — 30-хвилинний профіль навантаження (48 інтервалів на добу), ~20с/день через PLC
- **MR** — добові показники по тарифах T1-T4, ~5с/день
- **Energy** — поточні накопичувальні покази лічильника на момент запиту
- **Monthly** — зведення за місяць по тарифах

---

## Напрямки енергії

- **A+ споживання** — активна енергія з мережі (основний показник)
- **A- генерація** — зворотна активна енергія (сонячні панелі, генерація в мережу)
- **Q+ реактивна** — реактивна імпортована
- **Q- реактивна** — реактивна експортована

---

## PLC команди

```
LP (30-min):    A+ = CMD 21,  A- = CMD 83,  Q+ = CMD 72,  Q- = CMD 73
MR (daily):     A+ = CMD 22,  A- = CMD 79
Energy:         A+ = CMD 15,  A- = CMD 91
Monthly:        A+ = CMD 23,  A- = CMD 82
```

---

## Збереження в БД

Зчитані дані зберігаються в штатну БД SQLite (mtxdb.sdb):

- **LP** -> таблиця `archivo_mhora` (pType: 21=A+, 22=A-, 23=Q+, 24=Q-)
- **MR** -> таблиця `indicaciones_mtx1` або `indicaciones_mtx3` (pType: 1=A+, 2=A-)

Після збереження дані одразу видимі в графіках UI.

---

## Захист від перевантаження

- **Конкурентний лок** — одночасно лише 1 пряме зчитування (`/tmp/meter_archive.lock`)
- **Авто-чанкування** — фронтенд розбиває великі діапазони на частини (3 дні LP / 14 днів MR)
- **Серверний ліміт** — максимум 7 днів LP / 30 днів MR за один HTTP-запит
- **Не блокує PollReader** — використовує окремий лок, PollReader працює паралельно

## Інтерфейс (meter_ui.html)

- Вибір типу даних, напрямку, діапазону дат
- Пресети: 7 / 30 / 60 днів
- Живий таймер під час зчитування
- Лог прогресу по кожному чанку
- Кнопка "Стоп" для зупинки
- Масове зчитування кількох лічильників
- MR за замовчуванням (найчастіший запит)
- Wildcard маски в черзі (напр. `2506*`)
- Чекбокси для масових операцій

## Файли

- **meter_archive.php** — НОВИЙ. Серверна частина прямого PLC-зчитування з парсером бінарних відповідей та збереженням в БД.
- **meter_ui.html** — Оновлений. SPA інтерфейс — секція прямого зчитування, wildcard маски, чекбокси черги, таймер.

## Завантаження на RT4

```bash
# 1. Завантажити файли
scp www/rt2/meter_archive.php www/rt2/meter_data.php www/rt2/meter_ui.html www/rt2/status_api.php www/rt2/config_api.php root@192.168.100.100:/www/rt2/

# 2. Перезапустити php-fpm (при зависших воркерах)
ssh root@192.168.100.100 "systemctl restart php-fpm"

# 3. Перевірити файли
ssh root@192.168.100.100 "ls -la /www/rt2/meter_archive.php"
```

## API приклади

Добові показники A+ (споживання) за 2 дні, зберегти в БД:

```bash
curl "http://192.168.100.100:8000/rt2/meter_archive.php?serial=001A790D12263E6E&type=mr&dir=a+&save=1&date_from=2026-03-01&date_to=2026-03-02"
```

Генерація A- за тиждень:

```bash
curl "http://192.168.100.100:8000/rt2/meter_archive.php?serial=001A790D12263E6E&type=mr&dir=a-&save=1&date_from=2026-03-09&date_to=2026-03-15"
```

30-хв профіль за 1 день:

```bash
curl "http://192.168.100.100:8000/rt2/meter_archive.php?serial=001A790D12263E6E&type=lp&save=1&date=2026-03-15"
```

Поточні накопичувальні покази:

```bash
curl "http://192.168.100.100:8000/rt2/meter_archive.php?serial=001A790D12263E6E&type=energy"
```

Переглянути лог:

```bash
curl "http://192.168.100.100:8000/rt2/meter_archive.php?log=1"
```

## Параметри meter_archive.php

- **serial** — серійний номер лічильника, 16 hex символів (напр. `001A790D12263E6E`)
- **type** — тип даних: `lp`, `mr`, `energy`, `monthly`
- **dir** — напрямок енергії: `a+`, `a-`, `q+`, `q-` (за замовч. `a+`)
- **date** — одна дата у форматі `YYYY-MM-DD`
- **date_from** — початок діапазону `YYYY-MM-DD`
- **date_to** — кінець діапазону `YYYY-MM-DD`
- **save** — `0` або `1` — зберегти результат в БД
- **psw_level** — рівень пароля: `1`, `2`, `3` (за замовч. `3`)
- **log** — `1` — показати лог замість зчитування

## Технічні обмеження

- **Обладнання:** RT4 (STM32 OpenSTLinux), 64MB RAM
- **PHP:** 8.1.29, php-fpm 3-5 воркерів
- **БД:** SQLite3, WAL mode
- **PLC час відповіді:** LP ~20-120с/день, MR ~5-90с/день
- **Глибина архіву в лічильнику:** LP ~60 днів, MR ~12 місяців
- **Одночасних зчитувань:** 1 (конкурентний лок)
- **Max днів за запит:** LP 7 днів, MR 30 днів
